Requirements — Config Files First!

👦
Junior Dev
Why do we check config files first? Can't I just start coding?
🧑‍💻
Senior Dev
Trust me — skipping config checks costs you hours of debugging later. I learned the hard way! For this task we need 3 files to be correct:

📌  tsconfig.json → needs esModuleInterop: true
📌  package.json → needs [email protected] (NOT v2!)
📌  playwright.config.ts → needs timeout: 60000

The pdf-parse version is the sneaky one — v2.x is ESM-based and conflicts with "module": "commonjs" in your tsconfig. Always use v1.1.1.

Wait — does a TXT file also need a Translator like pdfParse?
🧑‍💻
Senior Dev
Great question! No! TXT file skips the Translator step completely.

PDF is like a book written in alien language — you need a translator to understand it.
TXT is like a book written in plain English — you can read it directly!

The only difference is you pass 'utf-8' to readFileSync for TXT and it returns a plain string — no pdfParse needed at all! ✅

TypeScript — Playwright Test
import { test, expect } from '@playwright/test';
import fs from 'fs';
const pdfParse = require('pdf-parse'); // require for CommonJS ✅

// ─── PDF FILE TEST ────────────────────────────────────────
test('Download and Validate PDF', async ({ page }) => {

  await page.goto('https://your-site.com/file-download/');

  // Step 2 — open hands AND click at the SAME time
  const [download] = await Promise.all([
    page.waitForEvent('download'),   // 🤲 open hands first
    page.locator('a.download-btn').click()  // 🖱️ then click
  ]);

  // Step 3 — save to disk (create folder if needed)
  if (!fs.existsSync('Downloads')) fs.mkdirSync('Downloads');
  const filePath = `Downloads/${download.suggestedFilename()}`;
  await download.saveAs(filePath);

  // Step 4 — check file exists ✅
  expect(fs.existsSync(filePath)).toBeTruthy();

  // Step 5 — Library Boy reads file → Reading Table (Buffer)
  const fileBuffer: Buffer = fs.readFileSync(filePath);

  // Step 6 — Translator converts 0s and 1s → English
  const pdfData = await pdfParse(fileBuffer);

  // Step 7 — Validate! ✅
  expect(pdfData.text).toContain('sample PDF');
  expect(pdfData.numpages).toBe(1);
  expect(pdfData.info.Author).toBe('Athan');
});

// ─── TXT FILE TEST ────────────────────────────────────────
test('Download and Validate TXT', async ({ page }) => {

  await page.goto('https://your-site.com/file-download/');

  const [download] = await Promise.all([
    page.waitForEvent('download'),
    page.locator('a.download-btn-txt').click()
  ]);

  if (!fs.existsSync('Downloads')) fs.mkdirSync('Downloads');
  const filePath = `Downloads/${download.suggestedFilename()}`;
  await download.saveAs(filePath);

  expect(fs.existsSync(filePath)).toBeTruthy();

  // Step 5 — Library Boy reads file → returns plain string!
  const text: string = fs.readFileSync(filePath, 'utf-8');
  //                                              ^^^^^^^^
  //                              Pass 'utf-8' → get string back directly
  //                              No pdfParse needed! Already human readable ✅

  // Step 6 — Validate directly! ✅
  expect(text).toContain('Hello');
  expect(text.trim()).not.toBeNull();
  console.log('TXT Content:', text);
});

Assertions — What Can We Validate?

👦
Junior Dev
I know we use expect() to validate — but what exactly can we assert on a PDF and TXT file? I only know toContain...
🧑‍💻
Senior Dev — Sameer
Great question! Think of assertions in 3 layers — File Layer, Content Layer and PDF Metadata Layer. Let me show you each one:

🗂️ Layer 1 — File Assertions (Both PDF & TXT)
TypeScript
// ✅ 1. Does the file EXIST on disk?
const fileExists: boolean = fs.existsSync(filePath);
expect(fileExists).toBeTruthy();

// ✅ 2. Does the file have the correct NAME?
const filename = download.suggestedFilename();
expect(filename).toBe('sample.pdf');
expect(filename).toContain('.pdf');     // check extension

// ✅ 3. Does the file have correct SIZE? (not empty)
const stats = fs.statSync(filePath);
expect(stats.size).toBeGreaterThan(0);    // file is not empty
expect(stats.size).toBeLessThan(5000000);  // file is under 5MB
📖 Layer 2 — Content Assertions (PDF & TXT)
TypeScript — PDF Content
const pdfData = await pdfParse(fileBuffer);

// ✅ Text contains a keyword
expect(pdfData.text).toContain('sample PDF');

// ✅ Text does NOT contain unwanted words
expect(pdfData.text).not.toContain('Error');
expect(pdfData.text).not.toContain('undefined');

// ✅ Text is not empty
expect(pdfData.text.trim().length).toBeGreaterThan(0);

// ✅ Text matches exact string
expect(pdfData.text.trim()).toBe('This is a sample PDF file.');

// ✅ Text matches a RegEx pattern
expect(pdfData.text).toMatch(/sample/i);   // case insensitive
expect(pdfData.text).toMatch(/\d{4}/);     // contains 4 digits
TypeScript — TXT Content
const text: string = fs.readFileSync(filePath, 'utf-8');

// ✅ Text contains a keyword
expect(text).toContain('Hello');

// ✅ Text does NOT contain unwanted content
expect(text).not.toContain('Error');

// ✅ Text is not empty
expect(text.trim()).not.toBe('');

// ✅ Line count check
const lines = text.split('\n');
expect(lines.length).toBeGreaterThan(0);

// ✅ Starts with expected text
expect(text.trim()).toMatch(/^Hello/);    // starts with Hello
🔍 Layer 3 — PDF Metadata Assertions (PDF only)
TypeScript — PDF Metadata
// ✅ Number of pages
expect(pdfData.numpages).toBe(1);
expect(pdfData.numpages).toBeGreaterThanOrEqual(1);

// ✅ PDF version
expect(pdfData.info.PDFFormatVersion).toBe('1.5');

// ✅ Author name
expect(pdfData.info.Author).toBe('Athan');
expect(pdfData.info.Author).not.toBe('');   // author not empty

// ✅ Creator tool
expect(pdfData.info.Creator).toContain('Word');

// ✅ No form fields (security check)
expect(pdfData.info.IsAcroFormPresent).toBe(false);
expect(pdfData.info.IsXFAPresent).toBe(false);

Key Takeaways — Never Forget

🏛️
Hard Disk = Library
Where all your files are stored. The Library never moves — it just holds the books.
📍
downloadFilePath = Shelf Number
The exact address of where your file lives. Without it the Library Boy gets lost!
👦
fs.readFileSync = Library Boy
Goes to the shelf, reads the raw data (0s and 1s) and hands it to the Reading Table.
📦
fileBuffer = Reading Table (PDF)
Holds raw 0s and 1s in memory. The book is on the table — but still in alien language!
🔤
string = Reading Table (TXT)
Pass 'utf-8' and you get back plain text directly. Already in English — no translator needed!
🔄
pdfParse = Translator (PDF only)
Converts 0s and 1s to English. Only PDF needs this step — TXT skips it completely.
🎯
Assert in 3 Layers
File layer (exists, size, name) → Content layer (toContain, toMatch) → Metadata layer (pages, author).
⚡
Promise.all = Two Hands at Once
Open your hands AND ask for the book at the SAME time — otherwise you will drop it!
✅
Config First — Always!
Check tsconfig, package.json and playwright.config BEFORE writing any code. Saves hours.
